Refined Architectural Design

1. Updated System Architecture
- Frontend Layer: Next.js (React) Dashboard providing live KPI visualizations and interactive simulation controls.
- API Layer: FastAPI handling REST requests, acting as the bridge between the UI and the simulation core.
- Simulation Core: SimPy Discrete Event Simulation (DES) managing the state of hospital wards, patient queues, and staff shifts.
- AI Decision Engine: Stable Baselines 3 PPO Model integrated directly into the simulation step loop to optimize patient admissions and bed allocations.

2. Workflow Diagram
[User/Admin] Configures hospital parameters via UI 
       --> [Frontend] Sends API request to reset simulation
               --> [FastAPI] Initializes SimPy Environment & loads AI Models
                       --> [SimPy] Generates hospital events (patient arrivals, discharges)
                               <--> [PPO Agent] Observes hospital state & predicts best routing actions
               <-- [FastAPI] Aggregates live metrics (queues, strain, HSI)
       <-- [Frontend] Polls metrics and updates live dashboard charts

3. Input-Process-Output
- Inputs: Simulation parameters (General Beds, ICU Beds, Staff counts, Arrival Rates), Scenario flags (Normal, Flu Surge, Mass Casualty).
- Process: DES environment processes stochastic patient arrivals. The RL Agent dynamically allocates resources based on real-time state vectors (occupancy rates, queue lengths, patient severity).
- Outputs: Time-series metrics including Hospital Strain Index (HSI), Department-level queue lengths, Staff Strain Ratio, and plain-text Strategic Insights generated by the LLM (or rule-based fallback).
